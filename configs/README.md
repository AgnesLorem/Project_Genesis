# Configs Folder

Purpose: Repository home for data-driven gameplay configuration.

This folder mirrors the documented conceptual `config/` hierarchy from `docs/CONFIG_GUIDE.md` and `docs/CONFIG_STRUCTURE.md`.

Current status: folder foundation only.

No active gameplay configuration data is added in this task.

## Rules

- Gameplay values must come from approved configuration.
- Config files must follow documented schemas.
- Reserved config surfaces must remain inactive unless approved.
- Do not add fake balance values or placeholder gameplay data as final content.
